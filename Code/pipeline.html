Step 1 — Open this file correctly

Browsers block localhost calls from a file:// page (CORS). You need to serve the HTML from a local server. Open your terminal, cd to the folder containing this file, and run:

python3 -m http.server 8080

Then open http://localhost:8080/pipeline_author.html in your browser. Keep this terminal running while you use the app.

Python is pre-installed on macOS and most Linux distros. On Windows, install from python.org then use the same command in PowerShell.

Step 2 — Start Ollama with CORS enabled

Ollama must be told to accept requests from your local server. Start it like this instead of the plain ollama serve:

# macOS / Linux OLLAMA_ORIGINS="*" ollama serve
# macOS — if Ollama runs as a menu bar app, run this first then restart it launchctl setenv OLLAMA_ORIGINS "*"
# Windows (PowerShell) $env:OLLAMA_ORIGINS="*"; ollama serve

Step 3 — Pull the recommended models

These cover all pipeline stages. Pull them once; they stay cached locally (~6 GB each for the 7b models).

ollama pull deepseek-r1:7b ollama pull qwen2.5-coder:7b ollama pull gemma3:4b ollama pull qwen3:4b ollama pull llama3:8b ollama pull phi3:mini

Step 4 — Configure & test

Default URL is http://localhost:11434. Only change this if Ollama is on a different port or machine.
